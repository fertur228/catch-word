# TakeWord — план: реальное распознавание + вырезка фона

> Обновлено: 2026-06-28. Источник по продукту — `../catchword_project_doc.md` (спека §11).
> Это план превращения мок-сканирования в настоящее: камера → реальное слово+перевод+IPA → вырезанный стикер → коллекция.
> Статус: **утверждён, к реализации**. Платежи отложены по решению. Ключ распознавания подключаем позже.

## TL;DR (выбранная архитектура)

**Гибрид: распознавание — в облаке, вырезка — на устройстве.** Оба шага идут параллельно за уже существующим экраном «Распознаю…».

- **Распознавание (облако):** один вызов мультимодальной модели из Supabase Edge Function (ключ только на сервере). Модель по фото возвращает JSON: слово на изучаемом языке + перевод на родной + IPA + рамку (bbox) + категорию.
- **Вырезка (на устройстве):** маленький нативный Swift-модуль вызывает iOS 17 `VNGenerateForegroundInstanceMaskRequest` → прозрачный PNG-стикер. Бесплатно, офлайн, без скачивания моделей.

Существующий поток `камера → scanning → result → save` **переиспользуется почти 1:1** — меняется не приложение, а пара вызовов функций. Главная по ценности правка — заменить эмодзи на реальный вырез на экране результата.

## Решение по модели распознавания (юнит-экономика)

Прочёсан рынок (Gemini, OpenAI, открытые VLM, спец-API, on-device). **Claude — самый качественный, но не самый дешёвый и без бесплатного тарифа.** Победитель по юнит-экономике:

| Модель | $/1000 сканов | Бесплатный тариф | Статус | Роль |
|---|---|---|---|---|
| **`gemini-2.5-flash-lite`** ⭐ | **~$0.21** ($0.10/$0.40 за 1M) | щедрый дневной, без карты | GA/стабильна (с июл 2025) | **дефолт** |
| `gemini-3.1-flash-lite` | ~$0.63 ($0.25/$1.50) | без карты | новее, OpenAI-слой «beta» | запасной (1 строка) |
| gpt-5-nano (OpenAI) | ~$0.21 | большой, но фото → в обучение OpenAI | GA | запасной (минус приватность) |
| Claude Haiku 4.5 | ~$2.3 | нет | GA | премиум-фолбэк на сложных кадрах |

**Дефолт — `gemini-2.5-flash-lite`.** Дешевле 3.1 в ~3×, стабильна, зрелый OpenAI-совместимый путь. 3.1 лучше «видит», но для бытовых предметов 2.5 «достаточно», а слабое место (IPA) у обеих одинаковое.

**Ключевой приём:** функция пишется в **OpenAI-совместимом** формате (эндпоинт `https://generativelanguage.googleapis.com/v1beta/openai/`, `image_url` data-URI + `response_format` json_schema). Тогда смена провайдера/модели — **одна строка** (`baseURL` + `model` + ключ), без релиза в App Store.

**Что это даёт:** для MVP распознавание фактически **бесплатно** (дневного бесплатного лимита хватает на десятки новых пользователей в день). Сверх лимита — ~$0.003 на пользователя (15 сканов). Деньги — не ограничение; ограничение — квоты и качество.

### Оговорки (выловлены проверкой)
1. **IPA от модели не доверять слепо** — для редких слов галлюцинирует. У нас уже есть встроенный словарь EN→RU с IPA (`src/lib/dictionary.ts`): для известных слов берём IPA из него, иначе — от модели.
2. **Точную рамку** брать с устройства (Apple Vision), а не из координат модели.
3. **Downscale: 1024px — это ДЛИННАЯ сторона** (иначе фото = 6 тайлов ≈ +50% к токенам).
4. В Edge Function — **retry/backoff на 429** (бесплатный тариф троттлит раньше номинала).
5. Точный бесплатный лимит **не зашивать как гарантию** — смотреть в Google AI Studio; не гарантируется.
6. **Пересмотреть 3.1** на платном объёме, если её качество окупит 3× цену.

## Фазы реализации

Ограничение: держать `npx tsc --noEmit` и `npx expo export -p ios` зелёными. **Фазы 0–1 — только JS, остаются зелёными.** Фаза 2 вводит нативный модуль → нужен `expo prebuild --clean` + `pod-install` + дев-сборка **на реальном iPhone** (подпись уже настроена, см. `SETUP.md`).

### Фаза 0 — водопровод: носим фото, а не слово-строку (только JS, ~0.5–1 день)
Снимаем реальное фото и проносим его + результаты через экраны через in-memory job-store, а не через URL-параметр `word`.
- Новый `src/lib/scan-job.ts` (или расширить `collection-context.tsx`): по `jobId` хранит `{ photoUri, result?, cutoutUri? }`.
- `src/screens/camera-screen.tsx`: `useRef<CameraView>` (`:206`), `onShutter` (`:144`) делаем `async`, гейт `tryScan()` (`:147`) оставляем; вместо мок-выбора (`:158-159` `getRandomRecognizable()`) — `takePictureAsync({quality:0.7})` и навигация с `{ jobId }`. Убрать импорт `getRandomRecognizable` (`:53`).
- `scanning-screen.tsx`: читать `jobId` вместо `word` (`:52`); таймер (`:79-85`) пока заглушка-таймер; добавить guard `navigated`.
- `result-screen.tsx`: данные из job, не из `getRecognizableByWord(params.word)` (`:91`); в `CardContent` (`:56-63`) добавить `imageUri?: string | null`.
- `src/components/sticker.tsx`: проп `imageUri?` + ветка `expo-image` (картинка, иначе эмодзи). Прокинуть в `word-tile.tsx:21`, `card-detail-screen.tsx:97/150`, `collection-screen.tsx:88`.

**Готово:** затвор делает реальное фото, на результате видно фото (полным кадром) как «стикер», слово ещё мок, сохранение работает. Сборка зелёная.

### Фаза 1 — реальное распознавание + реальный (обрезанный) стикер, БЕЗ нативного модуля (только JS, ~2–4 дня)
Реальное фото → реальное слово + картинка, целиком на телефоне, **без Swift**. Стикер — аккуратный кроп фото по bbox (нативный вырез придёт в Фазе 2).
- Зависимости (через `npx expo install`): `expo-image-manipulator` (resize/crop/base64), `expo-file-system` повысить до **прямой** зависимости (сейчас транзитивная).
- **Бэкенд (новое):**
  - `supabase/functions/recognize/index.ts` (Deno): CORS → `consume_scan()` RPC (402 при превышении) → кеш по `(image_hash, learning_lang, native_lang)` → вызов **Gemini 2.5 Flash-Lite** (OpenAI-совместимый, `response_format` json_schema) → валидация в коде → кеш → ответ `{objects, cached, scans_left}`. Ключ из `supabase secrets set GEMINI_API_KEY`. Модель из env `GEMINI_MODEL` (дефолт `gemini-2.5-flash-lite`).
  - `supabase/migrations/0001_recognize.sql`: таблицы `recognition_cache`, `scan_usage` + атомарный `consume_scan(device_id)` (бесплатный лимит = `FREE_SCAN_LIMIT`).
- **Клиент (новое)** `src/lib/recognize.ts`: ужать до ~1024px (длинная сторона) JPEG q0.6 + base64 одним `manipulateAsync`; md5; POST в функцию с anon-ключом; `toRecognizable(obj)` (drop-in вместо `getRandomRecognizable()`); `cropToSticker(photoUri, bbox)` → файл в documentDirectory. IPA: известные слова — из `dictionary.ts`, иначе из ответа.
- `collection-context.tsx`: генерировать/хранить анонимный `device_id` в `key_value` (рядом с `onboarded`, `:28-30`); серверный `scans_left` — источник правды.
- `scanning-screen.tsx` (`:79-85`): заменить таймер на `await recognize(...)` + мин. задержка ~900мс (чтобы анимация всегда играла); на успехе кроп bbox → `cutoutUri` в job → `router.replace('/result', { jobId })`. Ошибки: лимит → `/paywall`; нет сети/не распознал → назад на камеру с сообщением (скан не списываем).
- `result-screen.tsx` (`onSave :132-144`): `imageUri: content.imageUri ?? null`. БД менять НЕ нужно — колонка `image_uri` уже есть (`db.ts:42/149`, чтение `:111`). `onRetake` (`:151`) — назад на камеру.

**Готово:** навёл на кружку → реальное «mug / кружка / mʌɡ / Дом», реальный кроп как стикер, сохранение офлайн в скрапбук, рендер везде. Сборка зелёная, без prebuild.

### Фаза 2 — настоящая вырезка фона (нативный Swift Expo-модуль, ~2–4 дня)
Заменить кроп на настоящий прозрачный вырез. Единственная нативная часть.
- `npx create-expo-module@latest --local` → `modules/subject-lift/` (`modules/` не в gitignore и автолинкуется при prebuild — Swift переживёт регенерацию, в отличие от `ios/`).
- `modules/subject-lift/ios/SubjectLiftModule.swift`: `AsyncFunction("liftToPNG")(uri) -> file:// PNG`. iOS 17: `VNGenerateForegroundInstanceMaskRequest` → `generateMaskedImage(...)` → temp `.png` с альфой. Фон-очередь, `if #available(iOS 17.0, *)`, мягкий reject при «нет субъекта»/iOS<17, downscale больших CIImage, переиспользовать один CIContext.
- `expo-module.config.json`: `{ "platforms":["apple"], "apple":{ "modules":["SubjectLiftModule"] } }`. `src/index.ts` реэкспортит `liftToPNG`.
- `scanning-screen.tsx`: `liftToPNG(photoUri)` параллельно с `recognize(...)` (`Promise.all`); предпочесть вырез, иначе кроп Фазы 1, иначе эмодзи.
- Сборка: `npx expo prebuild --clean` → `npx pod-install` → дев-сборка **на реальном iPhone 14**.

**Готово:** на карточке — предмет с убранным фоном. Аккуратный фолбэк, если маску не получить.

## Самое быстрое «заработало на телефоне» — на этой неделе
**Фазы 0 + 1.** Полный цикл на реальном устройстве: фото → реальное слово+перевод+IPA → реальный (кроп) стикер → офлайн в коллекцию, лимит сканов, ключ на сервере. **Без нативного модуля и prebuild**, сборка зелёная. Хардкодим: пара языков en-US/ru-RU, главный объект, анонимный `device_id`, `gemini-2.5-flash-lite`. Фаза 2 потом меняет источник картинки одной строкой.

## Топ рисков
1. **Вырезка не работает в симуляторе/Expo Go** — только на живом iPhone (Vision-маска кидает на симуляторе by design). Поэтому сначала всё на JS (Фазы 0–1).
2. **«Функция не находится»** — почти всегда забыт `npx pod-install`. Весь Swift в `modules/`, не в `ios/`.
3. **Утечка ключа / расходы** — ключ только в секретах Supabase; фото ≤1024px; кеш по хешу; лимиты; не оставлять `--no-verify-jwt` после демо.
4. **Бесплатная подпись живёт 7 дней** — дев-сборку пересобирать раз в неделю.
5. **Не «сжигать» скан при ошибке** — лимит тратим только на успешное распознавание; нет сети/не распознал/вырез не вышел — возвращаем скан (локально и на сервере).
