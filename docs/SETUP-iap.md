## Как это всё работает (картинка в голове)

Смысл: **деньги за подписку на iPhone обязаны идти через Apple** (правило App Store).
Мы не берём карту сами — это делает Apple. RevenueCat — «прослойка», которая
проверяет чеки Apple, помнит кто что купил, и сообщает нашему серверу.

```
Пользователь жмёт «Купить» в приложении
        │
        ▼
RevenueCat SDK  ──►  Apple (StoreKit)  ──►  окно оплаты Apple, списание с Apple ID
        │
        ├─(1) МГНОВЕННО: RevenueCat говорит приложению «premium активен»
        │        → доступ открывается сразу, не дожидаясь интернета/сервера
        │
        └─(2) НАДЁЖНО: RevenueCat шлёт webhook на наш сервер (Supabase)
                 → сервер пишет строку в таблицу subscriptions
                 → премиум виден на сервере (лимит сканов) и на других устройствах
```

Ключевой трюк — **единый аккаунт**: при входе приложение делает
`Purchases.logIn(твой Supabase user.id)`. Поэтому «кто купил» в Apple = «кто ты» в
нашей базе = тот же ключ, что на вебе (Polar). Купил на iPhone → премиум есть и в
браузере, и наоборот. Это уже сделано в коде.

**Кто за что отвечает:**
- **App Store Connect (Apple)** — тут «товары» (подписки) и деньги/договор.
- **RevenueCat** — мозг: проверяет чеки, хранит подписки, шлёт вебхук.
- **Supabase** — наш сервер и база (принимает вебхук, хранит статус).
- **Приложение** — показывает пейволл и запускает покупку.

## Порядок работ (6 фаз)

Фазы 1→2→3→4 надо делать по порядку (каждая даёт данные для следующей). Фаза 0 —
предусловие, займись ей первой, т.к. одобрение банка/договора у Apple может идти
несколько дней.

---

### ФАЗА 0. Договор с Apple (БЕЗ этого IAP не работают вообще)

📍 **App Store Connect** → https://appstoreconnect.apple.com → **Business** (или
«Agreements, Tax, and Banking»).

1. Прими **Paid Apps agreement** (Платные приложения).
2. Заполни **Bank** (банковский счёт для выплат) и **Tax** (налоговые формы).
3. Дождись, пока статус «Paid Apps» станет **Active**.

⚠️ Пока «Paid Apps» не Active — подписки **не будут загружаться** в приложении,
хоть всё остальное настрой. Это самый частый ступор. Начни отсюда.

💡 Заодно: **Apple Small Business Program** (App Store Connect → Agreements) — если
твой доход < $1M/год, Apple берёт **15%** вместо 30%. Подай заявку, это бесплатно.

---

### ФАЗА 1. Создать подписки в App Store Connect

📍 **App Store Connect** → **My Apps** → **TakeWord** → в левом меню
**Subscriptions** (раздел Monetization / In-App Purchases).

1. **Создай Subscription Group** (группу). Название, напр. `TakeWord Premium`.
   (Группа нужна, чтобы месяц и год были «одной подпиской в разных периодах» —
   пользователь может переключаться между ними.)

2. **Добавь подписку №1 — месяц:**
   - Reference Name: `Premium Monthly` (это внутреннее имя, видно только тебе)
   - **Product ID: `com.almazbukayev.takeword.premium.monthly`** ← дословно!
   - Duration: **1 Month**
   - Price: **$6.99** (Apple сама пересчитает в другие валюты)
   - Localization (App Store Localization): Display Name `Premium (месяц)` + описание
   - **Review screenshot** — загрузи скрин пейволла (Apple требует для ревью)

3. **Добавь подписку №2 — год:**
   - Reference Name: `Premium Yearly`
   - **Product ID: `com.almazbukayev.takeword.premium.yearly`** ← дословно!
   - Duration: **1 Year**
   - Price: **$39.99**
   - **Introductory Offer** → тип **Free Trial** → длительность **7 дней** (это и есть
     «7 дней бесплатно» — Apple их обеспечивает, не мы)
   - Localization + review screenshot — так же

4. Статус каждой подписки будет **«Missing Metadata»**, пока всё не заполнишь →
   потом станет **«Ready to Submit»**. Так и должно быть — их отправим вместе с
   билдом (см. Фазу 6).

> Product ID **должны совпадать дословно** с тем, что впишешь в RevenueCat (Фаза 3)
> и что уже зашито в коде. Одна опечатка — подписка «не найдётся».

---

### ФАЗА 2. Ключ App Store Connect для RevenueCat

RevenueCat нужен ключ, чтобы проверять чеки Apple. Проще всего — **In-App Purchase key**.

📍 **App Store Connect** → **Users and Access** → вкладка **Integrations** →
**In-App Purchase** (ключи для покупок) → **+** (Generate).

1. Сгенерируй ключ, **скачай файл `.p8`** (даётся один раз — сохрани!).
2. Запиши рядом: **Issuer ID**, **Key ID** (показаны на той же странице).

Эти три вещи (`.p8` + Issuer ID + Key ID) понадобятся в Фазе 3.

---

### ФАЗА 3. Настроить RevenueCat

📍 https://app.revenuecat.com → зарегистрируйся (бесплатно до ~$2.5k/мес выручки).

1. **Create Project** → назови `TakeWord`.
2. **Add App** (внутри проекта) → выбери **App Store** → Bundle ID:
   **`com.almazbukayev.takeword`**.
3. В настройках этого App загрузи ключ из Фазы 2: **App Store Connect API Key**
   (`.p8` + Issuer ID + Key ID). Это включает валидацию чеков.
4. **Entitlements** (в меню слева) → **+ New** → identifier: **`premium`**.
   (Entitlement = «право доступа». Приложение спрашивает RevenueCat: активен ли
   `premium`? Код уже это делает.)
5. **Products** → **+ New** → добавь **два** product ID (дословно как в Фазе 1):
   - `com.almazbukayev.takeword.premium.monthly`
   - `com.almazbukayev.takeword.premium.yearly`
   Каждый → **привяжи к entitlement `premium`**.
6. **Offerings** → создай/открой **`default`** → добавь **2 пакета (packages)**:
   - Package **Monthly** → продукт `...premium.monthly`
   - Package **Annual** → продукт `...premium.yearly`
   (Offering — это то, что приложение показывает на пейволле. Код берёт `default`.)
7. **API Keys** (Project settings) → скопируй **публичный Apple/iOS ключ** — он
   выглядит как **`appl_XXXXXXXX`**. Пригодится в Фазе 4.
8. **Webhook**: Project settings → **Integrations** → **Webhooks** → **+ New**:
   - URL: `https://fkkloiiyzplpljlsyhyy.supabase.co/functions/v1/revenuecat-webhook`
   - **Authorization header**: придумай длинную случайную строку (напр. сгенерируй
     паролем-генератором) и вставь её сюда. **Запомни её** — то же значение пойдёт
     в Supabase в Фазе 4.

### ФАЗА 4. Вписать ключи в приложение и задеплоить сервер

Здесь два ключа из RevenueCat встают на места. Часть могу сделать я — скажи.

**A. Ключ в приложение (`.env`):**
```
EXPO_PUBLIC_REVENUECAT_IOS_KEY=appl_XXXXXXXX      # публичный ключ из Фазы 3.7
```
(В `.env.example` строка уже есть как подсказка.)

**B. Сервер Supabase — деплой (из терминала, в папке проекта):**
```bash
# 1) миграция — добавляет колонки под RevenueCat в таблицу subscriptions
supabase db push

# 2) сама функция-вебхук
supabase functions deploy revenuecat-webhook

# 3) секрет вебхука — ТА ЖЕ строка, что в Authorization в Фазе 3.8
supabase secrets set REVENUECAT_WEBHOOK_AUTH=<та-самая-строка>
```

**C. Пересобрать приложение**, чтобы новый ключ из `.env` попал в билд:
```bash
npx expo run:ios
```
(Expo Go не подойдёт — у нас dev-build под SDK 56. Это уже привычный запуск.)

После этого пейволл на iOS сам подтянет цены **из App Store** и кнопки станут
реально покупающими (вместо заглушки «Скоро»).

### ФАЗА 5. Проверить (до отправки в App Store)

**Быстро, локально (симулятор) — по желанию:**
- В Xcode можно сделать **StoreKit Configuration file** и тестировать покупки прямо
  в симуляторе, без реального App Store. Быстрый цикл разработки.

**По-настоящему (реальный сценарий) — Sandbox:**
1. 📍 App Store Connect → **Users and Access** → **Sandbox** → **Testers** →
   создай тестовый Apple ID (любой email, необязательно настоящий).
2. На **реальном iPhone** зайди в приложение, нажми «Купить».
3. Когда всплывёт окно Apple — войди этим **sandbox-тестером** (не своим настоящим
   Apple ID!). Деньги не спишутся — это песочница.

**Что проверить (чек-лист прохода):**
- [ ] На пейволле видны реальные цены и «7 дней бесплатно» на годовом
- [ ] После покупки премиум открывается **сразу** (лимит сканов пропал и т.п.)
- [ ] В Supabase → таблица `subscriptions` появилась строка с твоим `user_id`,
      `status=active` (или `trialing`), `store=revenuecat` — значит вебхук дошёл
- [ ] Кнопка **«Восстановить покупки»** возвращает премиум (удали приложение,
      поставь заново, восстанови)
- [ ] Отмена подписки (в sandbox период короткий) → премиум пропадает после истечения

---

### ФАЗА 6. Отправить в App Store

⚠️ **Главная готча (частый отказ 2.1):** при **первой** отправке подписки надо
приложить **к тому же билду**, что уходит на ревью. Иначе ревьюер не сможет
протестировать покупку → отказ.

1. В версии приложения (App Store Connect → версия) в разделе **In-App Purchases /
   Subscriptions** прикрепи обе подписки к этой версии.
2. Убедись, что статусы подписок — **«Ready to Submit»** (метаданные заполнены).
3. **EULA-ссылку добавь в поле Description** приложения (не только в приложении) —
   Apple это проверяет и может завернуть за отсутствие.
4. **App Review Notes**: укажи демо-аккаунт (логин/пароль) и что подписка
   тестируется через sandbox. Это ускорит ревью.
5. Отправь на ревью.

## Деньги — как это выглядит по факту

- Пользователь платит, скажем, $6.99. **Apple берёт свою комиссию** (30%, или **15%**
  если ты в Small Business Program — см. Фазу 0), остальное выплачивает тебе на банк
  из Фазы 0 (Apple платит раз в месяц с задержкой).
- **RevenueCat бесплатен** пока выручка < ~$2.5k/мес — на старте платить не надо.
- **7 дней бесплатно**: первые 7 дней списания нет; если не отменил — на 8-й день
  Apple спишет годовую цену. Приложение всё это время показывает премиум (`trialing`).

## Если что-то не работает — куда смотреть

| Симптом | Причина / что проверить |
|---|---|
| На пейволле нет цен, кнопка «Скоро» | Не задан `EXPO_PUBLIC_REVENUECAT_IOS_KEY`, или не пересобрал приложение, или продукты в ASC ещё «Missing Metadata», или «Paid Apps» не Active (Фаза 0) |
| «Продукт не найден» | Product ID в ASC ≠ в RevenueCat ≠ в коде. Должны совпадать дословно |
| Купил, но премиум не открылся | Продукт не привязан к entitlement `premium` в RevenueCat (Фаза 3.5) |
| Премиум есть на iPhone, но нет на сервере/вебе | Вебхук не дошёл: проверь URL и что `REVENUECAT_WEBHOOK_AUTH` в Supabase = Authorization в RevenueCat. Логи: RevenueCat → Webhooks → история доставки, и Supabase → Functions → revenuecat-webhook → Logs |
| Вебхук отвечает 401 | Секрет не совпадает (Фаза 3.8 ↔ Фаза 4.B шаг 3) |
